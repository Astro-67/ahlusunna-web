# Ahlusunna Information Architecture

## Overview

This document maps the complete navigation flow and page structure of Ahlusunna based on the seed.ts mock data layer. Every route connects logically to the data hierarchy: **Level → Subject → Course → Module → Lesson**.

---

## Data Hierarchy

```
Level (awali, kati, endelea)
└── Subject (quran, hadith, fiqhi, tawhidi, sira)
    ├── Course (quran-intro, quran-tajweed, hadith-intro, fiqh-purification, tawhid-meaning, sirah-birth)
    │   └── Module (quran-intro-m1, quran-intro-m2, hadith-intro-m1)
    └── Lesson (21 total published lessons)
```

## Navigation Connections

### Subject Detail Page (`/subjects/:slug`)

```
[← Nyuma] → /subjects
[Endelea →] → /lesson/:slug (next uncompleted lesson)
[Subject Lesson] → /lesson/:slug
[Other Subject Card] → /subjects/:otherSlug
```

### Lesson Page (`/lesson/:slug`)

```
[Breadcrumb]
  Nyumbani → /
  Masomo → /subjects
  Subject Name → /subjects/:slug
  Lesson Title → (current page)

[Prev →] → /lesson/:prevSlug (same subject, lower order)
[Next →] → /lesson/:nextSlug (same subject, higher order)

[Sibling Strip] → /lesson/:siblingSlug (all lessons in same subject)
[Mark Complete] → Updates user progress in localStorage
```

### Progress Page (`/progress`)

```
[Completed Lesson Card] → /lesson/:slug
[Empty State CTA] → /subjects
```

### Admin Dashboard (`/admin`)

```
[Users Table - Demote/Make Admin] → toggles role
[Contacts Table - Mark as Read] → updates contact status
[Admin Content] → /admin/content
```

## Access Rules

### By Level

| Level | Access Condition | isPublic | Routes |
|-------|------------------|----------|--------|
| `awali` (Beginner) | Everyone | `true` | `/subjects`, `/subjects/:slug`, `/lesson/:slug` |
| `kati` (Intermediate) | Login + 70% awali complete | `false` | `/subjects/intermediate` |
| `endelea` (Advanced) | Login + 70% kati complete | `false` | `/subjects/advanced` |

### By Role

| Role | Access |
|------|--------|
| `public` (guest) | `/`, `/subjects`, `/subjects/:slug`, `/about`, `/contact`, `/login`, `/register` |
| `learner` | All above + `/progress`, `/subjects/intermediate` (if unlocked), `/subjects/advanced` (if unlocked) |
| `moderator` | All above + `/moderator` |
| `admin` | All above + `/admin`, `/admin/content` |

### Level Unlock Logic (from seed.ts)

```typescript
// User unlocks kati after completing 70% of awali lessons
// User unlocks endelea after completing 70% of kati lessons

const beginnerLessons = lessons.filter(l => l.levelId === 'awali')
const completedBeginner = user.progress.filter(slug =>
  beginnerLessons.some(l => l.slug === slug)
).length
const beginnerProgress = completedBeginner / beginnerLessons.length

if (beginnerProgress >= 0.7) {
  user.levelAccess.push('kati')
}
```

## Prev/Next Navigation Logic

Lessons are navigated within the **same subject**, ordered by the `order` field:

```typescript
// In lesson.$slug.tsx
const siblings = allSeedLessons
  .filter(l => l.subjectId === lesson.subjectId && l.status === 'published')
  .sort((a, b) => a.order - b.order)

const currentIndex = siblings.findIndex(l => l.slug === slug)
const prev = currentIndex > 0 ? siblings[currentIndex - 1] : null
const next = currentIndex < siblings.length - 1 ? siblings[currentIndex + 1] : null
```

### Sample Navigation Chain (Fiqhi Subject)

```
/subjects/fiqhi
  ↓
/lesson/rules-of-purification (order: 1)
  Prev: none | Next: /lesson/rules-of-prayer
  ↓
/lesson/rules-of-prayer (order: 2)
  Prev: /lesson/rules-of-purification | Next: /lesson/rules-of-fasting
  ↓
/lesson/rules-of-fasting (order: 3)
  Prev: /lesson/rules-of-prayer | Next: none
```
